ISSUE:
------
- User checked in at 12:34 PM IST but system showed status as "present" instead of "late"
- Root cause: Django stores times in UTC, but check-in logic was comparing UTC time against 9:30 AM cutoff
- Example: 12:34 PM IST = 07:04 AM UTC, and 07:04 < 09:30, so system incorrectly marked as "present"

SOLUTION IMPLEMENTED:
--------------------
1. Added 'late' status to Attendance model choices
2. Updated check_in() function to convert UTC to Asia/Kolkata timezone before comparison
3. Installed pytz library for timezone handling
4. Created migration to update database schema
5. Created fix_late_arrivals.py script to recalculate existing records

HOW IT WORKS NOW:
-----------------
1. When employee checks in, system gets current time in UTC
2. Converts UTC time to Asia/Kolkata timezone (IST)
3. Compares local time against 9:30 AM cutoff
4. If check-in time > 9:30 AM IST → Status = 'late'
5. If check-in time ≤ 9:30 AM IST → Status = 'present'
6. HR receives notification for late arrivals
