                    TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════

ISSUE: Employee at office getting "network error"
SOLUTION: 
   - Check if office IP is added to ALLOWED_OFFICE_IPS
   - Verify employee is on Regus WiFi
   - Check AWS logs for actual IP: tail -f /var/log/gunicorn/error.log
   - Look for: "Access denied - Client IP: X.X.X.X"
   - Add that IP to ALLOWED_OFFICE_IPS

ISSUE: Admin getting error on check-in
SOLUTION:
   - Should be fixed now
   - Admin bypasses all restrictions
   - If still error, check logs

ISSUE: Employee with approved WFH cannot check-in from home
SOLUTION:
   - Verify WFH is approved (not pending)
   - Check WFH dates include today
   - Check in HR Dashboard → WFH Requests

ISSUE: System not detecting office IP correctly
SOLUTION:
   - Check if behind load balancer/proxy
   - System reads X-Forwarded-For header
   - Verify AWS/Nginx configuration
